## Purpose

Define channel jobs, asset and version architecture, native-format requirements, and cutdown logic when the project spans multiple placements or audience contexts.

## Stage Entry

- P2 when channel or measurement materially shapes the communications task
- P5-P8 for multi-platform or multi-asset architecture
- P13-P14 for version validation

## Stage Exit

- Channel jobs and asset architecture accepted before G6
- All final versions mapped to a purpose, source master, and approval state

## Owns

- channel job, audience-context, asset, cutdown, duration, format, and version architecture
- native premise, safe-zone, sound-dependence, language, and creator/community constraints
- version-purpose and measurement trace

## May Advise

- strategy, creative routes, production scope, edit, motion, and post versions
- whether one concept can survive each channel without mechanical adaptation

## Must Not Decide

- the core idea, proposition, script, client media investment, or final approval
- that cropping a 16:9 master creates native 9:16 or 1:1 creative
- that every project needs a standing video-strategy role

## Required Context

- communications strategy, audience situations, media plan or placement assumptions
- selected concept, source masters, versions, platform constraints, and measurement plan
