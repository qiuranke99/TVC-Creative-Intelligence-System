# TCIS V3 Decisions

## 2026-07-14 - Production skills are user-managed external extensions

- Decision: the 30-capability architecture is complete without bundling production
  skills. Skills listed by capability packages are resolved only when a routed step needs
  them and remain under the user's installation, creation, update, and compatibility
  control.
- Missing-skill behavior: identify the exact skill, purpose, affected step, whether that
  step must pause, and which independent work can continue; then let the user create,
  download, upgrade, skip, or replace it.
- Boundary: TCIS does not bundle, auto-install, pin, upgrade, imitate, or silently replace
  a missing skill and never claims its expected artifact succeeded.

## 2026-07-14 - Public repository is a V3-only product surface

- Decision: publish the repository as `TVC-Creative-Intelligence-System` with only the
  current V3 architecture, Runtime, capability packages, fixtures, tests, and bounded
  verification evidence.
- Privacy boundary: live `tasks/current/` client state remains local-only and is excluded
  from Git and the package manifest.
- Release boundary: the public release requires a clean reachable root commit, no prior
  TCIS architecture releases, and independent anonymous clone verification before the
  repository becomes public.

## 2026-07-13 - Current snapshot reaccepted after root-cause release audit

- Decision: accept the current manifest-bound L0-L3 receipt after independent
  multi-surface counterexample review and root repair.
- Accepted repairs: media parsers and inspector identity; revision, lock, and conflict
  invariants; terminal WAL, leases, and revision retention; locale-independent state;
  behavioral validation; generated cleanup and manifest containment; P11/P14 authority
  and timing; canonical L0-L6 taxonomy.
- Public-release rerun: `121/121` Node tests, `5/5` Runtime surfaces, `73/73` fixtures, `9/9`
  synthetic projects, and the standalone verifier pass.
- Boundary: local L0-L3 GO does not claim real-client validation, legal/claims or rights
  clearance, vendor or physical production, external blind review, final release, or
  commercial outcome.

## 2026-07-11 - V3 local Runtime accepted for L0-L3

- Decision: accept the zero-dependency Node Runtime/CLI, canonical state engine,
  professional router, two-phase human decision loop, capability registry and generated
  specialists, executable fixtures, and synthetic E2E as current implementation truth.
- Acceptance: `runtime_implementation = PASS`, `agent_capability_migration = PASS`,
  `acceptance_fixture_execution = PASS`, and `synthetic_e2e_validation = PASS`.
- Boundary: `real_project_validation = NOT_RUN` and
  `commercial_production_readiness = NOT_PROVEN`.

## 2026-07-11 - Independent release gate controls L0-L3 acceptance

- Decision: acceptance is valid only after closing direct-lock, malformed-media, schema
  differential, platform-evidence persistence, WAL/content publication, and
  package-integrity findings.
- Root controls: only `commitLock()` may persist LOCKED; confirmation binds a changed
  descendant and hash; selected media requires contained actual bytes plus structure;
  campaign platform evidence is persisted; schema invariants use canonical local
  semantics; ambient memories remain disabled.
- Boundary: bounded fixtures and synthetic routes do not become client, legal, rights,
  production, blind-review, or effectiveness evidence.

## 2026-07-11 - Internal autonomy without D0 or user API

- Decision: D0 is not a user-facing authorization stage. Standing local authority covers
  Codex custom agents, local files and tools, approved production skills, browser
  research, and native image generation.
- Boundary: explicit human authority remains mandatory for external communication or
  publication, purchases, credentials, rights, claims/legal clearance, bookings,
  material budget or schedule changes, safety, irreversible deletion, and release.

## 2026-07-11 - Human interaction at every decision-bearing artifact

- Decision: brief, diagnosis, strategy, creative brief, routes, core concept, TVC
  expression, script/board, visual direction, director/treatment, PPM, actual selects,
  offline, and release follow
  `AI proposals -> human advice/selection -> AI revision -> human lock/reopen`.
- Boundary: hashes, manifests, logs, and machine validation are automatic unless they
  reveal a consequence requiring human judgment.

## 2026-07-11 - Creative platform is conditional

- Decision: use `Core Creative Decision` as the universal decision point.
- Consequence: one-off TVCs normally lock a film concept or advertising idea. A campaign,
  creative, or brand platform exists only for genuine multi-execution scope and must prove
  generative capacity, invariants, variables, and governance.

## 2026-07-11 - Advertising craft leads; engineering supports

- Decision: professional advertising practice is the creative operating layer.
- Consequence: the default path is immersion, diagnosis, communications strategy, sharp
  brief, Copywriter/Agency Art Director paired creation, CD critique, incubation, film
  development, and client review. Engineering supports evidence, state, recovery, and
  verification without replacing creative judgment.

## 2026-07-11 - Agency creative and production interpretation are separate

- Decision: the default route reaches approved agency concept, script, and board before
  production pitch and director treatment.
- Consequence: Agency Art Director and Production Designer are separate; Agency Producer
  and Production-company Producer are separate; Editor and Post Producer are separate;
  Commercial Director owns production interpretation after the appropriate gate.
- Exceptions: director-led, animation, VFX-first, AI-native, direct-to-client, and
  integrated-studio routes require explicit scope, IP, fee, and approval boundaries.

## 2026-07-13 - Retired workflow payloads remain absent

- Decision: payloads from the retired workflow are not V3 decisions, evidence, recovery
  inputs, migration sources, or available project assets.
- Boundary: negative safety rules and role-confusion fixtures may reference the retired
  concept without preserving or restoring its payload.
