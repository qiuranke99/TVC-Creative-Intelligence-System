# TCIS v3 专业 TVC 阶段、角色与决策矩阵

状态：active  
日期：2026-07-11  
适用：从 Client Brief 到 Final Master 的完整 TVC 项目；小项目可以合并产物，但不得合并不同决策权

## 1. 使用规则

本矩阵解决四件事：

1. 每个正式产物由谁负责内容完整性；
2. 哪些角色可以贡献、挑战或批准；
3. 哪些角色在该阶段默认不应进入；
4. 人类在产物形成前必须决定什么。

`Owner` 是专业内容责任人，不等于付款方、版权持有人或最终批准者。`Reviewer` 可以提出专业意见，但不能以共识或平均分取代 named decision owner。

## 2. 权力类型

| 类型 | 含义 |
|---|---|
| `O` Owner | 对产物的专业完整性负责，必须签核或明确退回 |
| `C` Contributor | 受 owner 邀请提供素材、方案或 craft 输入 |
| `R` Reviewer/Challenger | 检查本专业的风险、逻辑或越权，不共同署名创作 |
| `A` Approver | 对商业、品牌、法律、预算、制作或发布后果有正式批准权 |
| `N` Not default | 默认不进入；只有变体或明确例外才参与 |

## 5. 不得混淆的岗位

### 5.1 Agency Art Director 与 Production Designer

Agency Art Director 与 Copywriter 从 creative brief 共同创造广告 idea、visual concept 和 agency board。Production Designer 在导演获选后，把导演愿景变成场景、布景、道具、材质和 art department 的执行系统。

同一人可以在小型或一体化项目中兼任，但系统必须记录两个不同身份、两个进入时点和两套批准责任。

### 5.2 Agency Producer 与 Production-company Producer

Agency Producer 代表委托/代理商侧管理 pitch、供应商、预算、批准和 client/agency 协调。Prodco Producer 代表承制方建立 bid、crew、实际生产、安全、保险、合同、供应商和交付。

二者都不拥有核心广告 idea。

### 5.3 Creative Director 与 Commercial Director

Creative Director 对 agency idea、script 和品牌创意完整性负责。Commercial Director 获选后对 treatment、表演、调度、镜头、声音和影片执行愿景负责。

导演可以提出能显著改善 concept/script 的 alternative，但只有 agency/client 接受并更新上游产物后才生效。

### 5.4 Editor 与 Post Producer/Supervisor

Editor 对叙事、节奏、素材选择和剪辑 craft 负责。Post Producer/Supervisor 对流程、预算、供应商、版本和交付负责。不得让同一自动角色既剪片又自证 technical delivery PASS。

## 6. 变体与合法例外

### 6.1 Director-led / Production-company-originated

导演可以在 P4-P7 提前成为 concept partner，但必须满足：

- 客户/agency 明确选择 director-led scope；
- 付费、IP、保密、竞标和修改权写清；
- 不让多个竞标导演免费完成 agency 的创意工作；
- 仍区分商业/品牌、strategy、creative、legal 和 production approval。

### 6.2 Animation

- Agency concept/script/board 仍是输入；
- animation director、visual development、character/world、editor 和 animatic 更早进入；
- styleframes、character、environment、motion test、animatic 都是 decision-bearing artifacts；
- CG camera/lighting 可替代独立 DP，但责任必须显式。

### 6.3 VFX / technical mechanism first

VFX supervisor、DP、PD 或 special-effects expert 可在 concept/script 阶段做 feasibility prototype。其身份是 consultant，除非合同另行授予 concept ownership。

### 6.4 AI-native / hybrid

- asset canon 与 shot contract 在生成前锁定；
- prompt specialist 只编译已批准意图；
- 用户批准实际图像/视频，不批准不可见 prompt；
- identity、product geometry、copy、claim、continuity、reference/likeness rights 和 disclosure 有独立门；
- editor 可像 animation 一样在 animatic/shot timing 阶段提前。

## 7. Transition gates

| Gate | 进入条件 | 不通过时 |
|---|---|---|
| G0 Brief Accepted | 目标、范围、decision owner、限制和未知已显式 | 留在 P0，不让 creative team 猜目标 |
| G1 Diagnosis Selected | 主解释有证据、反证和未知；客户事实已核 | 继续研究或降低结论置信度 |
| G2 Strategy Locked | 目标链、受众、品牌角色、proof、KPI 和 sacrifice 一致 | 重开 P1/P2 |
| G3 Creative Brief Accepted | Strategy、CD、客户外部约束分别确认 | 返回 P2/P3 |
| G4 Route Selected | 2-3 条成熟路线均 on-brief；CD 推荐明确 | revise、none 或 reopen brief |
| G5 Core Concept Locked | film engine、product role、emotion/end 足以判断 | 继续 P4/P5，不做高精 visual |
| G6 Production-ready Creative | script、agency board、claims、proof、ballpark 可行性、客户批准齐全 | 不得 production pitch |
| G7 Award | treatment、bid、scope、IP、schedule、award 书面成立 | 不得正式 preproduction |
| G8 PPM Greenlight | 关键执行、rights、安全、预算、schedule、contingency 已批准 | 不得 shoot/generate |
| G9 Offline Lock | 结构、exact copy、claim、brand/product role 已锁 | 不得无控制进入 finish |
| G10 Release | final actual masters、rights、legal、QC、versions 全部通过 | STOP，不得发布 |
